`cx` models trust as a spectrum rather than a binary state.

## Levels

- hypothesis (Track B, MCP)
- memory (notes)
- proof (Track A, bundle)

## Trust mapping

- source tree -> trusted
- notes -> conditional
- agent output -> untrusted_until_verified
- bundle -> trusted

## Why

Strict determinism is required for proof, while development still needs flexibility.

`cx` resolves this by:

- allowing flexibility
- recording provenance
- enforcing trust boundaries

## Rule

Flexibility must be explicit and recorded.

Proof must remain deterministic.
