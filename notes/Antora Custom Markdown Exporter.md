Write a custom Antora exporter that emits a separate Markdown file for each
canonical doc topic such as architecture, manual, release, repository, and
workflows.

This exporter should take the curated Antora site under `docs/` and produce a
parallel markdown-based publication format that preserves the existing doc
boundaries rather than flattening everything into one huge file.
