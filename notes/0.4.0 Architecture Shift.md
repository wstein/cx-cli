`cx` 0.4.0 marks a transition from a Repomix-adapter system to a kernel-owned architecture.

## What

The system now separates:

- render kernel (proof path)
- MCP (hypothesis generation)
- notes (cognition layer)
- CI/release (assurance layer)

Repomix is no longer the conceptual center. It becomes a compatibility reference.

## Why

A deterministic system cannot depend on a patched external tool as its proof authority.

Ownership of:
- rendering
- token accounting
- structured plans
- verification

must move inside `cx`.

## How

- introduce native render kernel
- preserve compatibility via tests and contracts
- treat Repomix as a temporary verification oracle
- evolve toward kernel + bounded extensions
