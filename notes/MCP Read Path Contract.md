The MCP read path must remain available as the primary live exploration workflow.

## What

When the workspace is still changing, operators and agents need a stable read-oriented path for:

- inspecting files
- planning bundles
- reading notes
- checking workflow guidance

This path must exist without requiring proof-path bundling first.

## Why

If live exploration disappears or silently weakens, users lose the intended Track B workflow and start forcing proof tools into hypothesis work.

## How

Preserve the MCP read path by keeping these expectations stable:

- read-oriented tools stay available by default
- planning and inspection work without note mutation authority
- MCP remains bounded by the repository trust model
- exploratory workflows do not claim proof they cannot provide

## Rule

MCP must remain the primary path for live exploration, not a degraded shadow of bundle workflows.
