`v0.4` is closed as a release line because the kernel-owned proof path, notes governance, contract versioning, shared handover model, overlap strictness, and scanner pipeline contract are now implemented and enforced by tests, CI, and documentation.

## Closed

- native proof-path rendering is the production runtime
- Repomix is no longer a runtime dependency
- shared handover is kernel-owned and style-aware
- note targets and gating are explicit
- overlap ownership can be enforced strictly
- scanner stages and bounded scanner selection are implemented

## Post-v0.4

Scanner evolution now belongs to post-`v0.4` work:

- additional core scanners
- richer scanner policy and CI rollout
- any future extension/plugin model above the current bounded seam

## Rule

Do not reopen `v0.4` with new architecture scope. Treat further scanner growth as the next line of work, not unfinished release closure.
