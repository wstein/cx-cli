# Repomix Bundles

`cx` extends [repomix](https://github.com/yamadashy/repomix) with the concept of a **bundle folder** — a directory that collects one or more repomix output files alongside binary assets (images, diagrams, etc.), a canonical manifest, and SHA-256 checksums.

---

## What is a Bundle?

A bundle is a plain directory with the following structure:

```
my-bundle/
├── repomix-output.xml      # repomix output (XML, JSON, or plain)
├── repomix-output-docs.xml # additional repomix output (optional)
├── architecture.png        # binary asset (optional)
├── notes.txt               # plain-text asset (optional)
├── manifest.json           # generated by cx bundle
└── SHA256SUMS              # generated by cx bundle
```

`cx bundle` scans the folder, classifies files as repomix outputs or assets, computes a streaming SHA-256 for each file, and writes `manifest.json` and `SHA256SUMS`. These generated files are idempotent — running `cx bundle` again on the same folder overwrites them with fresh values.

## Commands

### `cx bundle <path>`

Scan a bundle folder and write `manifest.json` + `SHA256SUMS`.

```sh
cx bundle ./my-bundle
cx bundle ./my-bundle --zip   # also create my-bundle.bundle.zip
cx bundle ./my-bundle --quiet # suppress informational output
```

**Options**

| Flag | Description |
|------|-------------|
| `--zip` | Create `<folder-name>.bundle.zip` next to the bundle folder |
| `-q, --quiet` | Suppress informational output |

**Generated files**

- `manifest.json` — machine-readable index of all files, SHA-256 digests, sizes, and line counts.
- `SHA256SUMS` — sha256sum-compatible checksums for all entries, sorted by path.

---

### `cx list <path>`

List source files recorded inside one or more repomix output files.

```sh
cx list ./my-bundle                     # list all repomix files in folder
cx list ./my-bundle/repomix-output.xml  # list a single repomix file
cx list ./my-bundle --plain             # one path per line (for piping)
```

**Options**

| Flag | Description |
|------|-------------|
| `--plain` | Print one path per line, no decoration |
| `-q, --quiet` | Suppress totals and headers |

---

### `cx init`

Scaffold a `repomix.config.json` and a `cx.json` in the current directory (or `--cwd <dir>`).

```sh
cx init               # creates repomix.config.json + cx.json
cx init --ts          # creates repomix.config.json + cx.ts
cx init --force       # overwrite existing files
cx init --cwd ./proj  # run in a specific directory
```

**Options**

| Flag | Description |
|------|-------------|
| `--ts` | Generate `cx.ts` instead of `cx.json` |
| `--force` | Overwrite existing config files |
| `--cwd <dir>` | Target directory (default: `process.cwd()`) |
| `-q, --quiet` | Suppress informational output |

**Generated files**

- `repomix.config.json` — standard repomix configuration.
- `cx.json` — cx project configuration (or `cx.ts` with `--ts`).

---

### `cx cleanup <path>`

Remove `manifest.json` and `SHA256SUMS` (and optionally a ZIP archive) from a bundle folder.

```sh
cx cleanup ./my-bundle               # remove manifest.json + SHA256SUMS
cx cleanup ./my-bundle --zip         # also remove my-bundle.bundle.zip
cx cleanup ./my-bundle --zip-name custom.zip  # remove a named ZIP
```

**Options**

| Flag | Description |
|------|-------------|
| `--zip` | Remove `<folder-name>.bundle.zip` |
| `--zip-name <name>` | Remove a specific ZIP file by name |
| `-q, --quiet` | Suppress informational output |

## Manifest Schema

`manifest.json` is written by `cx bundle` and follows the `cx-bundle/v1` format.

```jsonc
{
  "format": "cx-bundle/v1",
  "createdAt": "2026-04-09T17:00:00.000Z",   // ISO 8601
  "createdBy": "cx-cli",
  "repomixFiles": [                            // repomix output files
    {
      "path": "repomix-output.xml",           // posix relative path
      "size": 42730,                           // bytes
      "lines": 1234,                           // text files only
      "sha256": "abc123...",
      "isBinary": false
    }
  ],
  "assets": [                                  // images, binaries, extras
    {
      "path": "architecture.png",
      "size": 89210,
      "sha256": "def456...",
      "isBinary": true
    }
  ],
  "metadata": {}                               // reserved for extensions
}
```

### File entry fields

| Field | Type | Description |
|-------|------|-------------|
| `path` | `string` | Relative path inside the bundle, posix separators |
| `size` | `number` | File size in bytes |
| `lines` | `number?` | Line count (text files only) |
| `sha256` | `string` | SHA-256 hex digest |
| `isBinary` | `boolean` | Whether the file is treated as binary |

---

## SHA256SUMS

The `SHA256SUMS` file uses the standard `sha256sum` format:

```
<hash>  <relative-posix-path>
```

All entries are sorted alphabetically by path for deterministic output. Verify with:

```sh
sha256sum --check SHA256SUMS
```

---

## ZIP Archive

When `--zip` is passed to `cx bundle`, a `.bundle.zip` archive is created next to the bundle folder (not inside it). Files inside the archive use posix paths relative to the bundle root, sorted for deterministic ordering.

```
my-bundle.bundle.zip
  manifest.json
  SHA256SUMS
  repomix-output.xml
  architecture.png
  notes.txt
```

## Programmatic API

The adapter and command runners are exported from `cx-cli`:

```ts
import {
  scanBundleFolder,
  writeManifestAndSha,
  createZipFromBundleFolder,
  parseRepomixFile,
  cleanupBundleGeneratedFiles,
  defineConfig,
  type BundleManifest,
  type FileEntry,
  type RepomixEntry,
} from 'cx-cli';
```

### `scanBundleFolder(path, opts?)`

Returns a `BundleManifest` (not yet written to disk).

```ts
const manifest = await scanBundleFolder('./my-bundle', {
  includeHidden: false,
  exclude: ['node_modules/**'],
});
```

### `writeManifestAndSha(path, manifest)`

Writes `manifest.json` and `SHA256SUMS` to the bundle folder.

### `createZipFromBundleFolder(bundlePath, outputZipPath)`

Creates a streaming ZIP archive. No full-archive buffering.

### `parseRepomixFile(filePath)`

Returns `RepomixEntry[]` from a repomix XML or JSON output file.

### `cleanupBundleGeneratedFiles(path, opts?)`

Removes generated artefacts. Returns an array of removed file names.

## cx.json Configuration

```jsonc
{
  "$schema": "https://raw.githubusercontent.com/wstein/cx-cli/main/schema/cx.schema.json",
  "version": "1",
  "bundle": {
    "includeHidden": false,
    "exclude": ["node_modules/**", ".git/**"]
  }
}
```

Use `cx.ts` with `defineConfig` for editor autocompletion:

```ts
import { defineConfig } from 'cx-cli';

export default defineConfig({
  bundle: {
    includeHidden: false,
    exclude: ['node_modules/**', '.git/**'],
  },
});
```

## Repomix Integration

`cx init` generates a `repomix.config.json` pre-configured for XML parsable output. Generate repomix output before bundling:

```sh
# 1. Generate repomix output
npx repomix

# 2. Bundle the folder
cx bundle ./my-bundle

# 3. Inspect contained files
cx list ./my-bundle
```

For programmatic repomix use, consult the [repomix documentation](https://github.com/yamadashy/repomix).
