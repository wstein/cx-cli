`cx` now ships a curated documentation site with four cooperating surfaces:

- native proof path: `bundle`, `validate`, `verify`, `extract`
- live workspace path: `mcp`
- durable cognition path: `notes`
- adapter/oracle path: expert diagnostics and parity only

This Antora site is the front door for the curated documentation surface.
The legacy Markdown files in `docs/` remain available as migration references, but the curated navigation now lives here.

== Start Here

- xref:manual/index.adoc[Operator Manual] for the shortest route to commands and workflows.
- xref:manual/operating-modes.adoc[Operating Modes] for choosing between live MCP, bundles, and notes.
- xref:architecture/index.adoc[Architecture Overview] for the arc42-based architecture spine.
- xref:release/index.adoc[Release Overview] for release assurance, schema publishing, and v0.4 closure.

== Why this site exists

The repository already behaves like a governed documentation system rather than a casual Markdown folder.
Antora gives that system explicit navigation, page classes, breadcrumbs, reusable primers, and a clearer separation between onboarding, reference, architecture, workflows, and release material.

== Migration boundary

The curated site currently covers the canonical docs surface.
It does not flatten the `notes/` graph into site-native content.
That boundary is deliberate.
