= CX System Contracts
:page-layout: default

[.doc-origin]
Source companion: `docs/SYSTEM_CONTRACTS.md`

++++
<p>This document defines the operational contracts that sit under the mental model:</p>
<ul>
<li>the <strong>cognition contract</strong> for durable repository knowledge</li>
<li>the <strong>boundary contract</strong> for what must never mix across surfaces</li>
<li>the <strong>trust propagation model</strong> for how confidence moves through the system</li>
</ul>
<p>See: <a href="mental_model/">MENTAL_MODEL.md</a> for the canonical semantics of the CX triad, Track A vs Track B, policy tiers, and artifact lifecycle.</p>
<h2>Cognition Contract V1</h2>
<p><code>cx</code> now distinguishes between a note that is merely parseable and a note that is fit to become durable repository memory.</p>
<p>The minimum cognition contract is:</p>
<ol>
<li>a note must have a stand-alone summary paragraph</li>
<li>that summary must contain enough substance to route from quickly</li>
<li>template boilerplate must not survive into committed notes</li>
<li>the note must stay atomic enough for humans and agents to classify quickly</li>
<li>manifests must carry explicit cognition metadata instead of pretending every valid note is equally trustworthy</li>
</ol>
<p>Current enforcement is intentionally narrow and machine-checkable:</p>
<ul>
<li>summary required</li>
<li>summary must be non-trivial</li>
<li>untouched template prompts are rejected</li>
<li>note size limits remain enforced</li>
</ul>
<p>Current scoring is intentionally diagnostic, not philosophical. <code>cx</code> measures observable signals such as summary density, structural clarity, evidence links, note age, and code-drift pressure. It does not claim to prove that the idea itself is true.</p>
<p>The test strategy now treats cognition quality as an adversarial surface. Unit coverage exercises sparse, boilerplate, stale, and high-signal notes separately so future refactors do not collapse the distinction between parseable memory and durable repository knowledge.</p>
<p>Why this protects you: <code>valid note != good note</code>. The cognition contract narrows that gap by refusing obvious low-signal memory and by making the remaining quality signals explicit in manifests and CLI review.</p>
<h2>Boundary Contract</h2>
<p>The system stays trustworthy only if certain boundaries remain hard.</p>
<h3>Track Boundaries</h3>
<ul>
<li><strong>Track B</strong> may investigate, search, diagnose, and preserve hypotheses.</li>
<li><strong>Track B</strong> must not claim promotable artifact truth.</li>
<li><strong>Track A</strong> may freeze, validate, verify, and extract proof-grade artifacts.</li>
<li><strong>Track A</strong> must not pretend that unverified live state is already handoff-safe.</li>
</ul>
<h3>Surface Boundaries</h3>
<ul>
<li><strong>MCP</strong> may expose live workspace state, but CI must never treat MCP output as promotable proof.</li>
<li><strong>Bundles</strong> may carry note summaries and trust metadata, but they are the proof surface only after validation and verification.</li>
<li><strong>Notes</strong> may preserve reasoning, but they remain conditional knowledge until humans or later proof paths confirm them.</li>
<li><strong>CI</strong> may trust Track A outputs and governance reports, but it must not trust exploratory Track B state as release evidence.</li>
</ul>
<h3>Mutation Boundaries</h3>
<ul>
<li>default MCP sessions must not cross from analysis into mutation silently</li>
<li>note mutation requires an explicit trust decision</li>
<li>dirty local convenience bundles must not be mistaken for promotable artifacts</li>
</ul>
<p>Why this protects you: once boundaries blur, the system stops being a trust system and becomes a bag of smart features with no reliable handoff semantics.</p>
<h2>Trust Propagation Model</h2>
<p>Trust moves through <code>cx</code> asymmetrically:</p>
<p>The shorthand is:</p>
<ul>
<li><strong>Source tree: trusted</strong></li>
<li><strong>Notes: conditional</strong></li>
<li><strong>Agent output: untrusted until verified</strong></li>
<li><strong>Bundle: trusted</strong></li>
</ul>
<table>
<thead>
<tr>
<th>Surface</th>
<th>Trust level</th>
<th>Why</th>
</tr>
</thead>
<tbody><tr>
<td>Source tree selected from the VCS master list</td>
<td><strong>trusted</strong></td>
<td>It is the canonical input set for planning and later verification</td>
</tr>
<tr>
<td>Notes that pass governance</td>
<td><strong>conditional</strong></td>
<td>They are durable reasoning, but still claims about the system rather than proof of it</td>
</tr>
<tr>
<td>Agent outputs and live reasoning traces</td>
<td><strong>untrusted until verified</strong></td>
<td>They may be useful, but they are not handoff-safe evidence by themselves</td>
</tr>
<tr>
<td>Bundle artifacts that pass validation and verification</td>
<td><strong>trusted</strong></td>
<td>They are the proof surface frozen under manifest, checksum, and verification rules</td>
</tr>
</tbody></table>
<p>Manifest trust metadata exists so downstream automation can see that difference directly instead of inferring it from prose. Manifest traceability metadata now also points to the concrete audit path: <code>cx bundle</code> as the Track A command, <code>cx notes check</code> as the note-governance review gate, and <code>.cx/audit.log</code> as the MCP decision ledger for agent actions.</p>
<p>Why this protects you: bad reasoning should not gain the same standing as verified artifacts just because it was serialized into Markdown or emitted by an agent.</p>
<h2>What This Does Not Claim</h2>
<p>These contracts do <strong>not</strong> prove:</p>
<ul>
<li>that a note is factually correct</li>
<li>that a summary is complete</li>
<li>that an agent decision was optimal</li>
<li>that a repository never contains stale knowledge</li>
</ul>
<p>Those remain review and future-system concerns. The current contract is about refusing obvious low-signal cognition, keeping boundaries explicit, and propagating trust labels honestly.</p>
