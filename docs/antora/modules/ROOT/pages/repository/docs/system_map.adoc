<h2>Read It Left To Right</h2>
<ol>
<li>Run <code>cx mcp</code> when you need fast interactive help on live code.</li>
<li>Use <code>cx notes</code> to preserve reasoning that should survive the current session.</li>
<li>Run <code>cx bundle</code> when the work must become a reproducible artifact.</li>
<li>Run <code>cx verify</code> when later humans or CI need proof instead of trust-by-habit.</li>
</ol>
<h2>Choose The Entry Point</h2>
<ul>
<li>Start with <code>cx mcp</code> when you are still asking questions.</li>
<li>Move to <code>cx notes</code> when the reasoning should survive the current session.</li>
<li>Move to <code>cx bundle</code> when the work needs a frozen handoff.</li>
<li>Finish with <code>cx verify</code> when a later reader should receive proof instead of a claim.</li>
</ul>
