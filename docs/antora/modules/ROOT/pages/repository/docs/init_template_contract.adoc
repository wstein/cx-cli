= Init Template Contract
:page-layout: default

[.doc-origin]
Source companion: `docs/INIT_TEMPLATE_CONTRACT.md`

++++
<p><code>cx init</code> always generates the shared workspace bootstrap files plus a
language-aware <code>Makefile</code>. Some templates also specialize the default
authoring-oriented <code>cx-mcp.toml</code> overlay, and TypeScript currently adds a
second build-artifact overlay.</p>
<p>The universal generated files are:</p>
<ul>
<li><code>cx.toml</code></li>
<li><code>.editorconfig</code></li>
<li><code>Makefile</code></li>
<li><code>cx-mcp.toml</code></li>
<li><code>.mcp.json</code></li>
<li><code>.vscode/mcp.json</code></li>
<li><code>.claude/settings.json</code></li>
<li><code>.codex/settings.json</code></li>
<li><code>notes/</code> scaffolding</li>
</ul>
<h2>Capability Levels</h2>
<ul>
<li><code>basic</code>: workspace-aware <code>Makefile</code> plus the shared MCP overlay contract.</li>
<li><code>enhanced</code>: specialized authoring overlay and richer local quality targets.</li>
<li><code>advanced</code>: enhanced template plus separate MCP overlays for authoring and
build-artifact inspection.</li>
</ul>

<h2>Overlay Intent</h2>
<ul>
<li>Authoring overlay: live source exploration and editing in MCP sessions.</li>
<li>Build overlay: compiled or emitted artifacts that should stay separate from
the default authoring surface.</li>
</ul>
<p>If a repository uses nonstandard source or output layouts, edit the overlays
independently instead of broadening the default authoring view.</p>
