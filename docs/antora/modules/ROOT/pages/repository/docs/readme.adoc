<p><code>cx</code> publishes its curated canonical documentation site from <code>docs/antora/</code>.</p>
<p>This <code>docs/</code> directory remains the repository-local reference surface during the migration and still matters for contributors, release review, and link stability inside the repo. The published site is the primary front door; these Markdown files are the checked-in companion surface while the transition finishes.</p>
<p><code>cx</code> is documented as four cooperating surfaces:</p>
<ul>
<li>native proof path: <code>bundle</code>, <code>validate</code>, <code>verify</code>, <code>extract</code></li>
<li>live workspace path: <code>mcp</code></li>
<li>durable cognition path: <code>notes</code></li>
<li>adapter/oracle path: diagnostics and parity only</li>
</ul>
<p>Use the published Antora site for curated navigation, breadcrumbs, architecture structure, and workflow entrypoints. Use this directory when you need the repository-local Markdown companions.</p>
<p>Schema and coverage publishing policy lives in <a href="release_checklist/">RELEASE_CHECKLIST.md</a>, which keeps the public GitHub Pages host and release mirror aligned with the checked-in <code>schemas/</code> files and the successful <code>main</code> CI proof path.
Developer command conventions for <code>make test</code>, <code>make verify</code>, and <code>make release</code> live in the repository notes and the operator manual.
The MCP surface has a documented stable subset, but the broader integration layer still evolves conservatively. Use <a href="stability/">STABILITY.md</a> for the stable subset and <a href="agent_integration/">AGENT_INTEGRATION.md</a> for client-facing setup guidance.</p>
<h2>What Changed In 0.4.0</h2>
<p><code>cx</code> now explicitly operates as three primary cooperating surfaces:</p>
<ul>
<li>immutable snapshots with <code>cx bundle</code></li>
<li>live agent protocol with <code>cx mcp</code></li>
<li>durable knowledge with <code>cx notes</code></li>
</ul>
<p>Track B generates hypotheses. Track A generates proofs. Notes preserve durable reasoning between them.</p>
<p>The adapter/oracle seam remains outside the shipped proof path. It exists for
diagnostics, parity visibility, and reference-oracle comparison rather than
ordinary operator workflows.</p>
<p>0.4.0 also makes three release-level changes explicit:</p>
<ul>
<li>Vitest coverage is now the authoritative coverage-reporting lane for release assurance.</li>
<li>The MCP surface has a stable subset, but the broader integration layer still evolves conservatively.</li>
<li>Notes governance, cognition scoring, contradiction checks, and trust propagation are part of the operating contract rather than incidental implementation details.</li>
</ul>
