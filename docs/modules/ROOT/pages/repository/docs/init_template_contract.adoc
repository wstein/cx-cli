= Init Template Contract
:page-layout: default


`cx init` always generates the shared workspace bootstrap files plus a
language-aware `Makefile`. Some templates also specialize the default
authoring-oriented `cx-mcp.toml` overlay, and TypeScript currently adds
a second build-artifact overlay.

The universal generated files are:

* `cx.toml`
* `.editorconfig`
* `Makefile`
* `cx-mcp.toml`
* `.mcp.json`
* `.vscode/mcp.json`
* `.claude/settings.json`
* `.codex/settings.json`
* `notes/` scaffolding

=== Capability Levels

* `basic`: workspace-aware `Makefile` plus the shared MCP overlay
contract.
* `enhanced`: specialized authoring overlay and richer local quality
targets.
* `advanced`: enhanced template plus separate MCP overlays for authoring
and build-artifact inspection.

=== Overlay Intent

* Authoring overlay: live source exploration and editing in MCP
sessions.
* Build overlay: compiled or emitted artifacts that should stay separate
from the default authoring surface.

If a repository uses nonstandard source or output layouts, edit the
overlays independently instead of broadening the default authoring view.

=== Enhanced Templates

[[rust-template]]
==== Rust

The Rust template is designed for Cargo workspaces with a richer local
verification gate than the basic templates.

Generated targets:

* `make build` -> `cargo build --release`
* `make test` -> `cargo test`
* `make check` -> `cargo check`
* `make verify` -> `cargo fmt --check`, `cargo clippy --all-targets -- -D warnings`, `cargo test`, `cargo build --release`
* `make certify` -> `verify`

The generated `cx-mcp.toml` exposes `src/**`, `tests/**`, `benches/**`,
`Cargo.toml`, `Cargo.lock`, and `README.md`. It excludes `target/**` so
build artifacts do not leak into the default authoring view.

[[go-template]]
==== Go

The Go template is intended for module-aware repositories using
`go.mod`.

Generated targets:

* `make build` -> `go build ./...`
* `make test` -> `go test ./...`
* `make check` -> `go vet ./...`
* `make verify` -> `check + test + build`
* `make certify` -> `verify`

The generated `cx-mcp.toml` exposes `cmd/**`, `internal/**`, `pkg/**`,
`go.mod`, `go.sum`, and `README.md`. Adjust the overlay if your project
keeps important source outside those conventional directories.

[[elixir-template]]
==== Elixir

The Elixir template is optimized for Mix-based applications with a
source-first MCP overlay.

Generated targets:

* `make build` -> `mix compile`
* `make test` -> `mix test`
* `make check` -> `mix format --check-formatted`
* `make verify` -> `check + test + build`
* `make certify` -> `verify`

The generated `cx-mcp.toml` exposes `lib/**`, `test/**`, `mix.exs`,
`mix.lock`, and `README.md`. It excludes `_build/**`, `deps/**`, and
coverage output so MCP stays focused on authoring files by default.

[[crystal-template]]
==== Crystal

The Crystal template is designed for `shard.yml` workspaces with a
small, source-oriented MCP surface.

Generated targets:

* `make build` -> `shards build` when shard metadata exists, otherwise `crystal build src/main.cr --release`
* `make test` -> `crystal spec`
* `make verify` -> `test + build`
* `make certify` -> `verify`

The generated `cx-mcp.toml` exposes `src/**`, `spec/**`, `shard.yml`,
`shard.lock`, and `README.md`. It excludes dependency and build caches
such as `.shards/**`, `lib/**`, and `bin/**`.

[[zig-template]]
==== Zig

The Zig template is aimed at `build.zig`-driven repositories.

Generated targets:

* `make build` -> `zig build`
* `make test` -> `zig build test`
* `make verify` -> `test + build`
* `make certify` -> `verify`

The generated `cx-mcp.toml` exposes `src/**`, `build.zig`,
`build.zig.zon`, and `README.md`. It excludes `zig-cache/**`,
`.zig-cache/**`, and `zig-out/**` to keep the default MCP surface
source-first.

=== Advanced Template
